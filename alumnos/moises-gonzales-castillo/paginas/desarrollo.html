Desarrollo del
Sensor Cardíaco

Descubre el proceso técnico que convierte señales eléctricas del corazón en datos legibles, alertas y visualizaciones en tiempo real.

Proceso

¿Cómo funciona el sensor?

El sensor de ritmo cardíaco sigue una cadena de pasos precisos desde la captura de la señal hasta la presentación del dato final.

01

Detección Óptica (PPG)

El sensor utiliza tecnología de Fotopletismografía (PPG). Un LED verde emite luz hacia el tejido del dedo o muñeca; un fotodetector mide cuánta luz es absorbida por la sangre. Cuando el corazón late, el volumen sanguíneo aumenta y absorbe más luz, creando una señal periódica.

02

Amplificación y Filtrado de Señal

La señal cruda es muy débil (microvoltios), por lo que un amplificador operacional (OpAmp) la eleva. Luego, filtros pasa-banda (0.5–4 Hz) eliminan el ruido de baja frecuencia (movimiento) y alta frecuencia (interferencia eléctrica), dejando solo la señal cardíaca limpia.

03

Conversión Analógico-Digital (ADC)

La señal analógica pasa por un convertidor ADC de 12 bits integrado en el microcontrolador. Con una frecuencia de muestreo de 100 Hz se obtienen 100 muestras por segundo, suficiente para detectar con precisión cada pico R del ciclo cardíaco (onda QRS).

04

Algoritmo de Detección de Picos

El firmware implementa el algoritmo Pan-Tompkins para detectar los picos R en la señal digitalizada. Calcula el intervalo R-R (tiempo entre latidos consecutivos) y aplica la fórmula:
BPM = 60 000 ms ÷ promedio(R-R ms)
El promedio de los últimos 4 intervalos reduce lecturas erráticas.

05

Clasificación y Alertas

Con el BPM calculado, el sistema lo clasifica según rangos clínicos: Bradicardia (<60 BPM), Normal (60–100 BPM) y Taquicardia (>100 BPM). Activa LEDs de color, vibración o notificaciones push según el nivel de riesgo.

06

Transmisión y Visualización

Los datos se envían vía UART/Bluetooth al dispositivo de visualización. La interfaz web recibe el valor cada 2 segundos, actualiza el contador BPM, anima la onda ECG y cambia los colores de estado para facilitar la interpretación inmediata.

Hardware

Componentes del Sistema

💡

Sensor MAX30102

Módulo PPG con LED rojo (660 nm) e infrarrojo (880 nm) para medición de SpO₂ y frecuencia cardíaca con interfaz I²C.

🤖

Microcontrolador

Arduino UNO o ESP32 procesan la señal ADC, ejecutan el algoritmo Pan-Tompkins y gestionan la comunicación serial.

📡

Módulo Bluetooth

HC-05 o BLE integrado en ESP32 transmite los datos BPM al dispositivo de visualización inalámbricamente.

🖥

Interfaz Web

HTML5, CSS3 y JavaScript muestran los datos en tiempo real con animaciones, gráficas ECG y alertas visuales dinámicas.
