FASES DEL PROYECTO

Proceso de construcción

Desde la idea inicial hasta el prototipo funcional, documentamos cada etapa del desarrollo del SmartBell.

Fase 01 — Investigación
Definición del problema y arquitectura
Se analizaron las necesidades de seguridad en el hogar y se definió la arquitectura del sistema. Se eligió el ESP32 por su conectividad WiFi integrada, bajo costo y comunidad activa. Se diseñó el flujo: botón → ESP32 → MQTT → App.
Fase 02 — Hardware
Montaje del circuito en protoboard
Se montó el ESP32 en la protoboard de 830 puntos. Se conectó el botón pulsador al pin GPIO4 con resistencia pull-up interna activada por software. Se verificaron las conexiones de alimentación (3.3V y GND) antes de cualquier prueba.
Fase 03 — Firmware
Programación del ESP32 con Arduino
Se desarrolló el firmware en C++ usando el framework Arduino para ESP32. Se implementó la detección del botón con debounce por software, la conexión WiFi, el cliente MQTT y la reproducción del mensaje de audio mediante síntesis TTS.
Fase 04 — Comunicación
Integración con servidor MQTT
Se configuró un broker MQTT (Mosquitto) para el paso de mensajes entre el ESP32 y la aplicación. Se definieron los topics: smartbell/ring, smartbell/status y smartbell/message.
Fase 05 — Frontend
Desarrollo del panel web de control
Se construyó la interfaz web con HTML, CSS y JavaScript puro. Se integró la Web Audio API para síntesis de voz, la MediaDevices API para acceso a la cámara, y el sistema de notificaciones push del navegador.
Fase 06 — Pruebas
Testing y optimización del sistema
Se realizaron pruebas de latencia (promedio 12ms en red local), pruebas de reconexión WiFi automática, y pruebas de grabación/reproducción de mensajes de voz. Se optimizó el consumo de energía del ESP32 usando deep sleep entre eventos.
